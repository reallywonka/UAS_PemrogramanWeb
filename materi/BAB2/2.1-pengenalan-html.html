2.1 Pengenalan HTML5

HTML (HyperText Markup Language) adalah bahasa markup inti yang mendefinisikan struktur dan konten dari sebuah halaman web. HTML bukanlah bahasa pemrograman karena tidak memiliki kemampuan logika atau komputasi; fungsinya semata-mata adalah untuk menandai elemen konten agar dapat ditampilkan oleh browser.

Pilar Utama Pengembangan Sisi Klien (Front-End)

  • HTML: bertanggung jawab atas struktur dan semantik (makna) konten.
  • CSS: bertanggung jawab atas presentasi dan estetika visual (gaya).
  • JavaScript: bertanggung jawab atas interaktivitas dan perilaku dinamis.

Transisi dari HTML4 ke HTML5

Evolusi ini didorong oleh kebutuhan web modern akan multimedia, dukungan perangkat mobile, dan aksesibilitas yang lebih baik.


Tabel Perbandingan Evolusi:

Aspek Evolusi HTML4 (Era Web 2.0 Awal) HTML5 (Era Web Modern) Implikasi dan Keunggulan
Deklarasi DOCTYPE Sangat kompleks dan panjang Sederhana:
<!DOCTYPE html>		 Mempermudah penulisan, meningkatkan konsistensi rendering (Standard Mode).
Struktur Tata Letak Mengandalkan tag <div> dengan id seperti id="header", id="footer" Elemen Semantik Baru (
<header>, <nav>, <section>, <article>, <footer>)		 Meningkatkan makna struktural dokumen (semantik), yang sangat penting untuk SEO dan screen reader.
Multimedia Memerlukan plugin eksternal seperti Adobe Flash atau Silverlight. Dukungan native (bawaan) menggunakan tag <audio> dan <video>. Memutar media lebih cepat, hemat baterai, dan tidak perlu instalasi plugin tambahan.
Penyimpanan Data Hanya menggunakan Cookies dengan kapasitas sangat kecil (4KB). Memperkenalkan Web Storage (localStorage, sessionStorage) dan database lokal. Memungkinkan aplikasi web bekerja offline dan menyimpan data lebih besar secara lokal.
Grafis Mengandalkan gambar statis (JPG/GIF) atau Flash untuk animasi. Mendukung grafis vektor (SVG) dan elemen <canvas> untuk rendering 2D/3D. Memungkinkan pembuatan game berbasis web dan visualisasi data yang dinamis.
